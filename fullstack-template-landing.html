Deploy

From local to production
in minutes

Two deployment options ‚Äî both work out of the box. The Dockerfile bundles everything for maximum simplicity.

Option A ‚Äî Fully Self-Contained
The simplest path. PostgreSQL, Supabase Auth, and your app bundled into a single container. Add a volume for persistence.
  • 1 Click "Deploy on Railway" from the GitHub repo
  • 2 Add a Volume mounted at /data for persistent storage
  • 3 Done ‚Äî your app, database, and auth system are live
‚≠ê Recommended for Production
Option B ‚Äî Railway PostgreSQL
Use Railway's managed PostgreSQL for better reliability, backups, and scalability in production environments.
  • 1 Click "Deploy on Railway" from the GitHub repo
  • 2 Add a PostgreSQL service in your Railway project
  • 3 Set DATABASE_URL to the Postgres connection string
  • 4 App auto-detects external database ‚Äî no volume needed
